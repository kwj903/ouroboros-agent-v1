## C. 핵심 아키텍처

### C-1. 채팅 1턴 데이터 흐름

```
사용자 메시지
  → POST /chat  (services.py::run_chat_turn)
    → 세션 생성/로드
    → build_memory_context()       # 세션 요약 + 워크스페이스 상태 + 장기 기억
    → run_agent(interaction_mode="web")
      → build_system_prompt()      # 동적 시스템 프롬프트 (10개 규칙 기반)
      → create_response()          # model.py — OpenAI 호환 호출
      → execute tool calls
      → detect __PENDING_APPROVAL__
    → append_history() / update_workspace_state()
    → persist_tool_trace()
    → generate_memory_suggestions()
    → compact_history_if_needed()  # 12개 초과 시 LLM 요약 (최근 8개 유지)
  → 응답: {answer, action_id, action_required, action_summary, suggestions_created}
```

### C-2. 승인 (Approval) 흐름

```
1. 모델이 request_ 계열 툴 호출
2. 툴이 __PENDING_APPROVAL__<payload> 문자열 반환
3. agent.py가 감지 → awaiting_approval 상태로 조기 종료
4. 프론트엔드에 승인 요청 표시 (action_id, summary 포함)
5. 사용자가 승인 또는 거부
   → 승인: execute_pending_action() → 실제 실행 → 히스토리에 반영
   → 거부: mark_rejected() → 취소 메시지 표시
```

**Action 상태 전이**: `pending` → `executed` | `rejected` | `failed`

### C-3. 세션 저장 구조

```
~/.ouroboros/state/sessions/<session_id>/
├── history.jsonl         # 대화 히스토리 (user/assistant)
├── rolling_summary.md    # 롤링 요약 (히스토리 컴팩트용)
├── workspace_state.json  # 마지막 browsed/read/modified/created/deleted path
├── meta.json             # title, created_at, updated_at
└── tool_log.jsonl        # 툴 실행 로그

세션 ID 형식: YYYYMMDD_HHMMSS_<6-char-hex>
```

### C-4. LLM 호출 구조

```python
# app/model.py
client = OpenAI(api_key=ALL_API_KEY, base_url=ALL_BASE_URL)

def create_response(messages, tools=None):
    return client.chat.completions.create(
        model=ALL_MODEL, messages=messages, tools=tools, tool_choice="auto"
    )
```

- Groq, OpenRouter, Google GenAI 등 **OpenAI 호환 API**면 모두 사용 가능
- `ALL_BASE_URL` 기본값: `https://api.groq.com/openai/v1`

## D. 파일별 역할

### D-1. 백엔드 핵심 파일

#### `app/agent.py` — 에이전트 루프
- `run_agent()` — step-limited 메인 루프 (최대 `max_steps` 반복)
- `build_system_prompt()` — **10개 규칙** 기반 시스템 프롬프트 동적 생성 (기존 30개 → 최적화)
- `plan_tasks()` — 복잡한 요청을 3-5개 서브태스크로 분해 (LLM 호출)
- `tasks_to_batch_operations()` — task list를 `request_batch_operations` payload로 자동 변환
- `_extract_pending_approval()` — `__PENDING_APPROVAL__` 파싱
- `_format_pending_approval_message()` — 웹/CLI 분기 포맷팅
- `trace_record` dict — 전체 실행 추적 (steps, tool_calls, results, status, tasks, completed_tasks, auto_batch)

#### `app/tool_registry.py` — 툴 등록
- `TOOLS: dict[str, ToolFunc]` — 18개 툴 함수 매핑
- `TOOL_SCHEMAS` — JSON schema 정의 (18개)
- 하단에서 `EXPOSED_TOOL_NAMES` 기준으로 필터링 재정의

#### `app/tools/workspace_tools.py` — 워크스페이스 툴
- **읽기**: `list_dir`, `tree_view`, `read_file`(utf-8/utf-8-sig/cp949), `get_workspace_info`
- **승인 요청**: `request_search_files`, `request_write_file`, `request_create_file`, `request_replace_text_in_file`, `request_delete_path`, `request_batch_operations`
- **실행**: `execute_pending_action()`, `reject_pending_action()`
- **배치 특화**: `_normalize_batch_operations()` (create+write 병합), `_execute_batch_operations()` (continue_on_error 지원)
- **상수**: `EXCLUDED_DIR_NAMES = {".git", ".venv", "__pycache__", ".agent_state"}`
- **⚠ 매우 중요**: `create_pending_action()` 시 `session_id=get_current_session_id()` 필수

#### `app/approvals.py` — 승인 상태 관리
- JSON 파일 저장 (`STATE_DIR/pending_actions.json`)
- Action ID: UUID 기반 8자리 hex
- `create_pending_action()`, `mark_executed()`, `mark_rejected()`, `mark_failed()`
- `list_pending_actions()` — status `"pending"` 필터, `created_at` 정렬

#### `app/conversation_state.py` — 세션 관리
- `SessionState` dataclass — 세션 디렉토리 및 파일 관리
- `append_tool_log()`, `load_tool_logs(limit=50)`
- `delete_session_by_id()`, `rename_session_by_id()`
- `set_title_if_missing()` — 첫 사용자 메시지로 자동 제목

#### `app/memory_manager.py` — 메모리/히스토리
- `build_memory_context()` — 세션 요약 + 워크스페이스 상태 + 장기 기억 결합
- `compact_history_if_needed()` — 12개 초과 시 LLM 요약 (최근 8개 유지)
- `update_workspace_state_from_trace()` / `update_workspace_state_from_approval()`
- 임계값: `COMPACT_THRESHOLD=12`, `RECENT_HISTORY_LIMIT=8`

#### `app/tool_trace_manager.py` — Tool trace / panel
- `persist_tool_trace()` — trace를 `tool_log.jsonl`에 저장
- `build_tool_panel()` — `last_note_search`, `last_note_read`, `last_file_read` 추출
- 휴리스틱 파싱: `_parse_search_notes_result()`, `_parse_read_text_result()`

#### `app/model.py` — LLM 클라이언트
- `create_response()` — OpenAI 호환 `chat.completions.create` 호출
- Groq / OpenRouter / Google GenAI 모두 `ALL_BASE_URL`로 전환 가능

#### `app/settings.py` — 환경변수 로드/검증
- `~/.ouroboros/.env` → 프로젝트 `.env` 순으로 로드
- 필수 변수 검증 (`ALL_API_KEY`), 선택 변수 기본값 제공
- `DEFAULT_OUTPUT_MODE`, `DEFAULT_RESPONSE_LANGUAGE` 검증

#### `app/paths.py` — 경로 정의
- `OUROBOROS_HOME = ~/.ouroboros` (기본)
- `STATE_DIR`, `LOGS_DIR`, `NOTES_DIR`, `CACHE_DIR`, `WORKSPACE_ROOT`
- `ensure_ouroboros_dirs()` — 자동 디렉토리 생성

#### `app/api/main.py` — FastAPI 라우트
- CORS: `localhost:3000`, `127.0.0.1:3000`, `localhost:5173`, `127.0.0.1:5173`
- `frontend/dist/` static serving

#### `app/api/services.py` — API 서비스 로직
- `run_chat_turn()` — 채팅 1턴 오케스트레이션
- `approve_action_api()` — 승인 실행 (session_id 필수, 누락 시 500)
- `reject_action_api()` — 승인 거부
- 세션/메모리 CRUD 래퍼
- **⚠ 코드 냄새**: `_is_action_required` 동일 구현 2번 정의

#### `app/api/schemas.py` — Pydantic 모델
- `ChatRequest`, `ChatResponse`, `CreateSessionResponse`, `RenameSessionRequest`
- `MemoryCreateRequest`, `MemoryUpdateRequest`

#### `app/long_term_memory.py` — 장기 기억

- `search_memory_records()` — lexical match + importance/recency 보너스

| `--reload` | 코드 변경 시 재시작 | off |
| `--dev` | Vite dev 서버 동시 실행 (HMR) | off |
| `--build` | 웹 실행 전 프론트엔드 빌드 강제 | auto |
| `--frontend-port` | 프론트엔드 dev 포트 | `5173` |
| `--no-open` | 브라우저 자동 열림 방지 | off |
| `--frontend-host` | 브라우저에서 열 호스트 | `localhost` |
