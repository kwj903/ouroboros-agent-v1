# 프론트엔드 UI/UX 개선 계획

## 목표
기존 기능을 유지하면서 사용자 경험, 가독성, 심미성을 최적화하여 ChatGPT 수준의 인터페이스 구현

## 현재 상태 분석
- 프론트엔드: React + TypeScript + Vite
- 주요 컴포넌트: 단일 App.tsx 파일 (약 1200줄)
- 스타일링: CSS 파일 (index.css)
- 문제점:
  1. 승인 요청이 우측 사이드바에 고정되어 가시성 낮음
  2. 긴 텍스트가 컨테이너를 overflow하여 잘림
  3. 사이드바를 모두 닫았을 때 채팅 영역이 과도하게 확장되어 가독성 저하
  4. 토글 버튼 UI가 직관적이지 않음
  5. 채팅 메시지 디자인이 현대적이지 않음

## 개선 방안

### 1. 승인 워크플로우 개선 (우선순위: 높음)
- **문제**: 우측 사이드바에 승인 목록이 있어 사용자가 놓치기 쉬움
- **해결책**:
  - 승인 요청 발생 시 채팅 하단에 배너 알림 표시 (숫자 + 텍스트)
  - 배너 클릭 시 중앙 모달 다이얼로그 표시
  - 모달 내부에서 작업 상세 정보 확인 및 승인/거부 버튼 제공
  - 외부 클릭 또는 ESC 키로 모달 닫기
  - 우측 사이드바에는 간략한 승인 개수만 표시 (클릭 시 모달 열기)

### 2. 채팅 인터페이스 개선 (우선순위: 높음)
- **문제**: 현재 메시지 디자인이 오래되었음
- **해결책**:
  - 말풍선 형태 메시지 UI 구현
  - 사용자 메시지: 오른쪽 정렬, 파란 계열 배경
  - 에이전트 메시지: 왼쪽 정렬, 회색 계열 배경
  - 메시지 전송 시 로딩 점 애니메이션 인디케이터 표시
  - Enter 키 = 전송, Shift+Enter 키 = 줄바꿈 지원

### 3. 레이아웃 및 반응형 디자인 (우선순위: 중간)
- **문제**: 사이드바를 모두 닫았을 때 채팅 영역이 화면 전체를 차지하여 가독성 저하
- **해결책**:
  - 채팅 영역 최대 너비 제한 (예: 800px)
  - 사이드바를 모두 닫았을 때 채팅 영역이 중앙에 위치하고 양쪽 여백 생성
  - ChatGPT와 같은 중앙 집중형 레이아웃 구현

### 4. 텍스트 오버플로우 방지 (우선순위: 높음)
- **문제**: 파일 경로, 메모리 내용 등이 컨테이너를 벗어남
- **해결책**:
  - 모든 텍스트 컨테이너에 적절한 줄바꿈 및 단어 끊김 처리
  - 긴 텍스트가 들어가는 영역에 최대 높이 설정 및 스크롤바 추가
  - 세션 제목/ID가 길 경우 말줄임 처리 + 호버 시 전체 내용 툴팁 표시

### 5. 디자인 시스템 개선 (우선순위: 중간)
- **문제**: 현재 스타일이 일관성 없음
- **해결책**:
  - CSS 변수를 이용한 디자인 토큰 시스템 구현
  - 일관된 색상 팔레트, 타이포그래피, 간격 규칙 적용
  - 버튼 hover/active 상태 명확화
  - 카드 디자인 표준화 (둥근 모서리, 적절한 그림자)

## 구현 순서

### Phase 1: 승인 워크플로우 개선
1. 승인 모달 컴포넌트 구현
2. 채팅 하단 알림 배너 추가
3. 승인 요청 시 알림 표시 로직 추가
4. 우측 사이드바 승인 목록을 간략한 개수 표시로 변경
5. 검증: 승인 요청 생성 → 알림 표시 → 모달 열기 → 승인/거부 → 모달 닫기 흐름 테스트

### Phase 2: 채팅 인터페이스 개선
1. 메시지 말풍선 스타일 구현
2. 로딩 인디케이터 추가
3. Enter/Shift+Enter 키 핸들링 구현
4. 검증: 메시지 전송 → 말풍선 표시 → 로딩 인디케이터 표시 테스트

### Phase 3: 레이아웃 및 반응형 디자인
1. 채팅 영역 최대 너비 제한 구현
2. 사이드바 닫음 상태에서 중앙 정렬 로직 추가
3. 검증: 사이드바 토글 시 채팅 영역이 중앙에 위치하고 적절한 너비 유지 확인

### Phase 4: 텍스트 오버플로우 방지
1. 모든 텍스트 컨테이너에 줄바꿈 및 단어 끊김 처리 적용
2. 긴 텍스트 영역에 최대 높이 및 스크롤바 추가
3. 세션 제목/ID에 말줄임 처리 + 호버 시 툴팁 추가
4. 검증: 다양한 길이의 텍스트 입력 시 오버플로우 없이 정상 표시 확인

### Phase 5: 디자인 시스템 개선
1. CSS 변수를 이용한 디자인 토큰 시스템 구현
2. 일관된 색상 팔레트, 타이포그래피, 간격 규칙 적용
3. 버튼 hover/active 상태 명확화
4. 카드 디자인 표준화
5. 검증: 전체적인 시각적 일관성 및 현대적인 느낌 확인

## 검증 기준
- 모든 기존 기능이 정상 작동해야 함 (툴 호출, 메모리 관리, 세션 관리 등)
- 승인 워크플로우가 직관적으로 작동해야 함
- 채팅 인터페이스가 현대적이고 사용하기 쉬워야 함
- 텍스트 오버플로우가 발생하지 않아야 함
- 사이드바 토글 시 레이아웃이 안정적이어야 함
- 반응형 디자인이 다양한 화면 크기에서 작동해야 함

## 주의사항
- 기존 기능을 변경하지 않고 UI/UX만 개선
- 단계별로 구현 후 각 단계별 검증 수행
- CSS 변경 시 기존 클래스 이름을 유지하여 JSX 구조 변경 최소화
- 가능하면 기존 로직을 재사용하고 확장하는 방식으로 구현
